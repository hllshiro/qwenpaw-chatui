## 验收标准

### 通知位置与显示

| ID | 描述 | 测试类型 | 前置条件 | 预期结果 |
|----|------|----------|----------|----------|
| AC-001 | 通知面板显示在右下角 | UI interaction | 触发任意通知 | 面板固定在视口右下角，距离底部和右侧各 16px |
| AC-002 | 通知面板进入动画 | UI interaction | 触发新通知 | 面板从右下角滑入并淡入，动画时长 300ms |
| AC-003 | 通知面板离开动画 | UI interaction | 关闭最后一条通知 | 面板向右下角滑出并淡出，动画时长 200ms |

### 多消息聚合

| ID | 描述 | 测试类型 | 前置条件 | 预期结果 |
|----|------|----------|----------|----------|
| AC-004 | 多消息只显示一个弹窗 | UI interaction | 同时存在 3 条通知 | 只显示一个通知面板，不堆叠 |
| AC-005 | 显示消息序号和总数 | UI interaction | 存在 3 条通知，当前查看第 2 条 | 右上角显示 "2/3" |
| AC-006 | 左右箭头切换消息 | UI interaction | 存在 3 条通知，当前查看第 2 条 | 点击左箭头显示第 3 条，点击右箭头显示第 1 条 |
| AC-007 | 第一条消息左箭头置灰 | UI interaction | 当前查看最新消息（第 1 条） | 左箭头显示为禁用状态，点击无效 |
| AC-008 | 最后一条消息右箭头置灰 | UI interaction | 当前查看最旧消息（最后一条） | 右箭头显示为禁用状态，点击无效 |
| AC-009 | 新消息自动切换到最新 | UI interaction | 当前查看第 2 条，收到新消息 | 自动切换到新消息，序号变为 1/4 |
| AC-010 | 关闭消息后显示下一条 | UI interaction | 存在 3 条通知，关闭当前第 2 条 | 自动显示剩余消息中的下一条，更新为 1/2 |
| AC-011 | 所有消息关闭后面板消失 | UI interaction | 存在 1 条通知，关闭该通知 | 通知面板消失 |

### 智能体完成通知

| ID | 描述 | 测试类型 | 前置条件 | 预期结果 |
|----|------|----------|----------|----------|
| AC-012 | 显示会话名称 | UI interaction | 智能体完成生成，会话名为 "测试会话" | 通知显示 "会话：「测试会话」" |
| AC-013 | 5 秒后自动消失 | UI interaction | 触发智能体完成通知 | 5 秒后通知自动关闭 |
| AC-014 | 点击跳转按钮导航到会话 | UI interaction | 触发智能体完成通知，点击 "跳转" | 页面跳转到对应会话页面 |
| AC-015 | 点击关闭按钮关闭通知 | UI interaction | 触发智能体完成通知，点击 "关闭" | 通知关闭 |

### 审批通知

| ID | 描述 | 测试类型 | 前置条件 | 预期结果 |
|----|------|----------|----------|----------|
| AC-016 | 显示严重级别和工具名 | UI interaction | 收到 HIGH 级别 execute_command 审批 | 通知标题显示 "HIGH \| execute_command" |
| AC-017 | 详情默认折叠 | UI interaction | 收到审批通知 | 详情区域默认折叠，显示 "▶ 详情" |
| AC-018 | 点击展开详情 | UI interaction | 点击 "▶ 详情" | 展开显示 findingsSummary 和 toolParams |
| AC-019 | 点击折叠详情 | UI interaction | 详情已展开，点击 "▼ 详情" | 详情区域折叠 |
| AC-020 | 点击批准按钮 | UI interaction | 收到 pending 状态审批，点击 "批准" | 调用 /api/approval/approve，状态变为 approved |
| AC-021 | 点击拒绝按钮 | UI interaction | 收到 pending 状态审批，点击 "拒绝" | 调用 /api/approval/deny，状态变为 denied |
| AC-022 | 审批通知不自动消失 | UI interaction | 收到审批通知，等待 10 秒 | 通知仍然显示，未自动关闭 |
| AC-023 | 已处理审批显示状态 | UI interaction | 审批已批准或拒绝 | 通知显示 "✅ 已批准" 或 "❌ 已拒绝" |
